# MimiSpec 0.3 Independent Usability Gate

The `0.3.0-rc.1` release workflow is intentionally blocked until
`tests/usability/0.3/trial-manifest.json` satisfies
`mimispec.usability/0.3`.

This gate does not prevent a clearly labelled `0.3.0-dev` technical snapshot.
As of 2026-07-20 the manifest is `in_progress` with 0/5 authors, 0/25
documents, 0/5 domains, and 0/4 five-minute successes. Technical corpora and
AI-assisted reverse transcriptions do not count toward these totals.

Required evidence:

- at least five authors who did not implement the parser;
- at least 25 final `.mms` documents across five domains;
- at least four authors create their first valid file within five minutes;
- every final file parses Complete, is lossless-exact, and semantic-round-trips;
- no open P0/P1 issue;
- every closed confirmed issue names an existing regression fixture.

Use:

```bash
mimispec usability check tests/usability/0.3/trial-manifest.json
mimispec usability check --require-complete
```

The first command reports progress. The second is the release-blocking gate.
Author identifiers must be pseudonymous, and committed documents must contain
no private or production data. If an author cannot license the original file,
commit a minimized anonymized fixture and record it as issue evidence instead.
